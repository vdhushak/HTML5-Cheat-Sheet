Basic HTML

The <!DOCTYPE html> declaration must be the very first thing in your HTML document, before the tag. It is an instruction to the web browser about what version of HTML the page using.

The <html> tag will be openned right after the doctype declaration. This tag tells the browser that this is an HTML document, it represents the root of an HTML document. The <html> tag is the container for all other HTML elements, except for the <!DOCTYPE> declaration. You can add the lang attribute, lang stands for language, to this tag to define the language of this page. Like so: <html lang="en">, where en stands for english.

Right after openning the <html> tag, you'll add the <head> tag. The <head> element is a container for all the head elements, those including a title for the document, scripts, cascading style sheets, meta information, and more. Everything you insert inside the <head> tag won't be rendered inside the document.

The <title> tag is required in all HTML documents and it defines the title of the document. It defines the title in the browser toolbar, it provides a title for the page when it is added to favorites and it displays a title for the page in search-engine results.

All the other tags that can go inside the <head> tag will be covered later in this tutorial.

Before starting with all the other tags, you'll need to close the <head> tag, with </head>, and then open the <body> tag. This tag defines the body of the document, and will contain all the the tags that can and will be rederend in the document, giving a layout to your webpage!

Semantics

Each tag for Semantics has a determined meaning a use, so be careful when chosing your tags for your layout!

<header>

The <header> tag represents a container for introductory content or a set of navigational links, and it usually contains/may contain one or more headings (h1-h6), the logo or icon of the website, and authorship info.

The <main> tag specifies the main content of a document, and it's content should be unique to the document. It should not contain any content that is repeated across documents such as sidebars, navigation links, copyright information, site logos, and search forms.

<div>

The <section> tag defines sections in a document, such as chapters, headers, footers, or any other sections of the document

The <div> tag defines a division or a section in an HTML document. It is often used as a container for other HTML elements to style them with CSS or to perform certain tasks with JavaScript.

<div>

The <article> tag specifies independent, self-contained content. An article should make sense on its own and it should be possible to distribute it independently from the rest of the site.

The <aside> tag defines some content aside from the content it is placed in. The aside content should be related to the surrounding content.

Programming

There's a tag <script> that you can add anywhere inside the <body> tag on your html code. Inside that tag you can add any client-side script in JavaScript. But that will also be covered in a future tutorial so I won't be getting in any details in this tutorial. You can also add a parameter src to, instead of writing JavaScript inside your HTML code, adding a .js file, and if you do so you can add that tag to the <head> too.

The <noscript> tag defines an alternate content for users that have disabled scripts in their browser or have a browser that doesn't support script. This tag can be used in both inside the <head> tags and <body> tags. When used inside the <head> element: <noscript> must contain only <link>, <style>, and <meta> elements. The content inside the <noscript> element will be displayed if scripts are not supported, or are disabled in the user's browser

The <link> tag defines a link between a document and an external resource. The <link> tag is commonly used to link to external .css files, and it is usually place inside the <head> tag.

Finally, there is the <meta> tag that provides metada to you website. Metadata is data (information) about data. Metadata will not be displayed on the page once it is normally within the <head> tag. Meta elements are typically used to specify page description, keywords, author of the document, last modified, and other metadata. The metadata can be used by browsers (how to display content or reload page), search engines (keywords), or other web services. The tags are usually are like this: <meta charset="UTF-8"> this line defines the charcterset for your webpage;
<meta name="description" content="Free Web tutorials"> this line gives a description about your website;
<meta name="keywords" content="HTML,CSS,XML,JavaScript"> this line gives some keywords to your website;
<meta name="author" content="John Doe"> this line gives authorship to the website;
<meta name="viewport" content="width=device-width, initial-scale=1.0"> and this line gives instructionsto the browser on how to control the page's dimensions and scaling.
